## Основной контент

### Что такое PASS24.online

**PASS24.online** — это облачная платформа (SaaS) для автоматизации управления доступом на территорию различных объектов: жилых комплексов, коттеджных поселков, бизнес-центров и промышленных объектов.

Платформа представляет собой комплексное решение, которое включает:
- Мобильное приложение для жителей и арендаторов
- Веб-интерфейс для службы охраны
- Панель администратора для управляющих компаний
- Облачный сервер для хранения данных и обработки запросов

### Архитектура решения

Платформа построена по модели SaaS (Software as a Service), что означает:
- **Облачное хранение данных** — вся информация хранится в защищенном облаке OnCloud.ru
- **Доступ через интернет** — работа с системой возможна с любого устройства с доступом в интернет
- **Автоматические обновления** — новые функции и исправления внедряются автоматически без участия клиента
- **Масштабируемость** — система автоматически масштабируется под нагрузку

## Ключевые возможности платформы

### 1. Мобильное бюро пропусков

**Описание:** Основная функция платформы — дистанционная выдача временных и постоянных пропусков через мобильное приложение.

**Возможности:**
- **Заказ пропусков в несколько кликов** — житель или арендатор может заказать пропуск для гостя прямо из мобильного приложения
- **Временные пропуска** — настройка времени действия пропуска (от нескольких часов до нескольких дней)
- **Постоянные пропуска** — для регулярных посетителей (родственники, няни, курьеры)
- **Мгновенная выдача** — пропуск выдается автоматически без участия охраны
- **История пропусков** — полный архив всех выданных пропусков с возможностью поиска

**Преимущества:**
- Экономия времени жителей (не нужно идти к охране)
- Снижение нагрузки на службу охраны
- Круглосуточная доступность (можно заказать пропуск в любое время)
- Полный контроль над доступом гостей

### 2. Электронный журнал событий

**Описание:** Полная замена бумажных журналов на электронный архив всех событий доступа.

**Возможности:**
- **Автоматическое ведение журнала** — все события фиксируются автоматически
- **Хранение в облаке** — данные хранятся безопасно и доступны в любое время
- **Поиск и фильтрация** — быстрый поиск событий по различным параметрам:
  - По дате и времени
  - По пользователю
  - По типу события (проход, проезд, выдача пропуска)
  - По точке доступа
- **Экспорт данных** — возможность экспорта журнала в различные форматы для отчетности
- **Неограниченный объем** — хранение данных без ограничений по времени

**Преимущества:**
- Отказ от бумажных журналов и их ведения
- Быстрый поиск нужной информации
- Невозможность потери или порчи данных
- Соответствие требованиям законодательства по хранению данных о доступе

### 3. Мгновенные уведомления

**Описание:** Система push-уведомлений о событиях доступа в режиме реального времени.

**Возможности:**
- **Уведомления о прибытии гостей** — житель получает уведомление, когда гость прибыл на объект
- **Уведомления о проходе/проезде** — информация о всех событиях доступа
- **Настройка уведомлений** — возможность настроить, какие события отслеживать
- **История уведомлений** — все уведомления сохраняются в приложении

**Преимущества:**
- Контроль доступа в режиме реального времени
- Возможность быстро отреагировать на неожиданные визиты
- Повышение безопасности объекта

### 4. Аналитика посещаемости

**Описание:** Детальная аналитика всех событий доступа с возможностью формирования отчетов.

**Возможности:**
- **Статистика посещаемости** — количество визитов, пиковые часы, популярные точки доступа
- **Аналитика по пользователям** — активность конкретных жителей или арендаторов
- **Аналитика по гостям** — частота визитов гостей, популярные временные интервалы
- **Визуализация данных** — графики и диаграммы для наглядного представления
- **Экспорт отчетов** — формирование отчетов для управляющей компании

**Преимущества:**
- Понимание паттернов посещаемости
- Оптимизация работы службы охраны
- Данные для принятия управленческих решений

## Способы идентификации

Платформа поддерживает несколько современных способов идентификации пользователей и гостей.

### QR-коды

**Описание:** Генерация уникальных QR-кодов для каждого пропуска.

**Как работает:**
- При заказе пропуска генерируется уникальный QR-код
- QR-код отправляется гостю в виде изображения
- Гость предъявляет QR-код на точке доступа
- Система сканирует код и проверяет его валидность

**Преимущества:**
- Не требует специального оборудования у гостя
- Работает на любом смартфоне
- Высокая скорость проверки
- Невозможность подделки (код генерируется динамически)

**Применение:**
- Временные пропуска для гостей
- Разовые визиты
- События с большим количеством посетителей

### NFC (Near Field Communication)

**Описание:** Использование технологии бесконтактной связи для идентификации.

**Как работает:**
- Гость подносит смартфон с NFC к считывателю
- Система идентифицирует пользователя по NFC-метке
- Доступ предоставляется автоматически

**Преимущества:**
- Быстрая идентификация (мгновенно)
- Удобство использования (просто поднести телефон)
- Высокая безопасность

**Требования:**
- Смартфон с поддержкой NFC
- Считыватель NFC на точке доступа

### Face ID (распознавание лица)

**Описание:** Биометрическая идентификация по лицу.

**Как работает:**
- При регистрации пользователя создается биометрический шаблон лица
- При проходе камера распознает лицо
- Система сравнивает с шаблоном и предоставляет доступ

**Преимущества:**
- Максимальная безопасность
- Удобство (не нужно ничего доставать)
- Быстрая идентификация

**Требования:**
- Камера с поддержкой распознавания лиц
- Согласие пользователя на обработку биометрических данных

### Bluetooth (BLE)

**Описание:** Использование Bluetooth Low Energy для идентификации через мобильное приложение PASS24.Key.

**Как работает:**
- Пользователь открывает приложение PASS24.Key на смартфоне
- Приложение устанавливает Bluetooth-соединение с точкой доступа
- Система идентифицирует пользователя и предоставляет доступ

**Преимущества:**
- Не нужно доставать телефон из кармана
- Работает на расстоянии (до нескольких метров)
- Низкое энергопотребление

**Применение:**
- Постоянные пользователи (жители, арендаторы, сотрудники)
- Управление доступом через турникеты, домофоны, шлагбаумы

## Интеграция с голосовыми помощниками

### Яндекс Алиса

**Описание:** Возможность заказа пропусков голосовыми командами через умную колонку Яндекс.

**Как работает:**
- Пользователь активирует навык PASS24.online в Яндекс Алисе
- Голосовой командой запрашивает пропуск для гостя
- Алиса обрабатывает запрос и создает пропуск
- Пропуск выдается автоматически

**Примеры команд:**
- "Алиса, закажи пропуск для Ивана на завтра с 10 до 18"
- "Алиса, когда придет курьер?"
- "Алиса, покажи мои активные пропуска"

**Преимущества:**
- Максимальное удобство для пользователей
- Возможность заказа пропуска, не отвлекаясь от других дел
- Современный и инновационный подход

**Требования:**
- Умная колонка Яндекс с Алисой
- Настроенный навык PASS24.online

- Sigur (российская система контроля и управления доступом)

- Другие системы через API

**Возможности интеграции:**
- Синхронизация данных между системами
- Единое управление доступом
- Совместное использование оборудования

### Интеграция с системами видеонаблюдения

**Поддерживаемые системы:**

## Часто задаваемые вопросы (FAQ)

### Вопрос 1: Нужно ли устанавливать какое-то оборудование на объекте?

**Ответ:** Для базовой работы платформы (выдача пропусков, электронный журнал) дополнительное оборудование не требуется. Система работает через мобильное приложение и веб-интерфейс. Для интеграции с точками доступа (турникеты, шлагбаумы) может потребоваться дополнительное оборудование, которое подбирается индивидуально под объект.

### Вопрос 2: Что происходит с данными, если интернет пропадет?

**Ответ:** Мобильное приложение может работать в офлайн-режиме для просмотра истории и заказа пропусков. Заказанные пропуска будут синхронизированы с сервером при восстановлении связи. Для работы точек доступа требуется стабильное интернет-соединение. Рекомендуется использовать резервный канал связи для критически важных объектов.

### Вопрос 3: Можно ли использовать систему без мобильного приложения?

**Ответ:** Да, веб-интерфейс позволяет выполнять все основные функции: заказ пропусков, просмотр журнала, управление пользователями. Однако мобильное приложение значительно удобнее для жителей и арендаторов.

### Вопрос 4: Сколько времени хранятся данные в электронном журнале?

**Ответ:** Данные хранятся без ограничений по времени. Вы можете настроить политику хранения данных в соответствии с требованиями вашего объекта и законодательства.

### Вопрос 5: Можно ли интегрировать платформу с существующей СКУД?

**Ответ:** Да, платформа может работать как надстройка над существующей СКУД. Поддерживается интеграция с системами Sigur, Bolid и другими через API. Возможности интеграции зависят от конкретной системы и обсуждаются индивидуально.

### Вопрос 6: Что делать, если гость забыл телефон или не может показать QR-код?

**Ответ:** В таких случаях служба охраны может проверить пропуск через веб-интерфейс по ФИО гостя или номеру пропуска. Также можно настроить резервные способы идентификации (например, по документу с последующей проверкой в системе).

## Типичные ошибки и их решение

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Пропуск не работает при сканировании QR-кода | Пропуск истек или был отменен | Проверить срок действия пропуска в системе, при необходимости выдать новый |
| Не приходят push-уведомления | Уведомления отключены в настройках приложения или устройства | Проверить настройки уведомлений в приложении и настройках устройства |
| Не могу зайти в веб-интерфейс | Неверные учетные данные или проблемы с интернетом | Проверить логин и пароль, проверить интернет-соединение |
| Данные не синхронизируются между устройствами | Проблемы с интернет-соединением | Проверить интернет-соединение, обновить страницу или перезапустить приложение |

## Критерии успеха

**Успешное внедрение платформы определяется следующими критериями:**

- Все пользователи зарегистрированы в системе и имеют доступ к мобильному приложению или веб-интерфейсу

- Система работает стабильно без сбоев
- Данные синхронизируются между всеми устройствами в реальном времени
- Пользователи довольны удобством использования системы

## Глоссарий терминов
