## Краткое описание

Данный документ описывает процесс приема и регистрации обращений клиентов в службу технической поддержки PASS24.online. Содержит пошаговую инструкцию по регистрации обращений, присвоению уникальных идентификаторов и первичной обработке входящих запросов.

## Основной контент

### Каналы приема обращений

Обращения в службу технической поддержки могут поступать следующими способами:

1. **Электронная почта:** support@pass24online.ru
2. **Онлайн-форма:** https://b24-6s4knr.bitrix24.site
3. **Телефон:** +7(495) 324-24-42

**Важно:** Все обращения должны быть зарегистрированы в системе обработки обращений независимо от канала поступления.

### Требования к обращению

Для более оперативного решения вопросов, обращение должно включать следующую информацию:

- Описание проблемы и пошаговое описание действий по воспроизведению проблемы (по возможности)
- Модель телефона, версия ОС телефона, в котором наблюдается проблема
- Номер используемой версии программного продукта
- Дополнительно, службой технической поддержки может быть запрошена другая информация необходимая для выявления и ликвидации проблем

## Пошаговая инструкция

### Шаг 1: Получение обращения

**Что делать:**
1. Проверить входящие сообщения во всех каналах связи
2. Зафиксировать время поступления обращения
3. Определить канал поступления обращения

**Где:** 
- Почтовый ящик support@pass24online.ru
- Система обработки обращений (для формы)
- Телефонная линия +7(495) 324-24-42

**Результат:** Обращение получено и зафиксировано

---

### Шаг 2: Регистрация обращения в системе

**Что делать:**
1. Создать новое обращение в системе обработки обращений
2. Заполнить обязательные поля:
   - Контактные данные клиента
   - Описание проблемы/вопроса
   - Канал поступления обращения
   - Дата и время поступления
3. Сохранить обращение

**Где:** Система обработки обращений (CRM Битрикс24)

**Результат:** Обращение зарегистрировано в системе

---

### Шаг 3: Присвоение уникального идентификатора (ID)

**Что делать:**
1. Система автоматически присваивает уникальный идентификатор (ID) обращению
2. Зафиксировать ID обращения
3. Убедиться, что ID отображается в заголовке обращения

**Где:** Система обработки обращений

**Результат:** Обращению присвоен уникальный идентификатор

---

### Шаг 4: Отправка уведомления клиенту

**Что делать:**
1. Отправить клиенту автоматическое уведомление о регистрации обращения
2. В уведомлении указать:
   - Уникальный идентификатор (ID) обращения
   - Информацию о начале обработки обращения
   - Рекомендацию сохранить ID для последующей переписки

**Где:** Система обработки обращений (автоматическая отправка)

**Результат:** Клиент получил уведомление с ID обращения

---

### Шаг 5: Проверка полноты информации

**Что делать:**
1. Проверить наличие всей необходимой информации в обращении:
   - Описание проблемы
   - Версия продукта
   - Информация об устройстве (если применимо)
   - Контактные данные клиента
2. Если информации недостаточно, подготовить запрос дополнительной информации

**Где:** Система обработки обращений

**Результат:** Определена полнота информации в обращении

## Особые случаи

### Работа с почтовыми обращениями

**Проблема:** Возможны проблемы с доставкой почтовых сообщений из-за работы сторонних почтовых сервисов или спам-фильтров.

**Решение:**
- Проблема считается принятой только если клиент получил письмо о регистрации с уникальным номером тикета
- Это означает, что письмо прошло проверку на антиспам и было зарегистрировано в системе поддержки
- В случае возникновения проблем с доставкой почтовых сообщений рекомендуется перенести решение проблем в чат-бот или звонок

### Работа с телефонными обращениями

**Что делать:**
1. Принять звонок
2. Выяснить суть проблемы
3. Зарегистрировать обращение в системе
4. Сообщить клиенту ID обращения
5. При необходимости запросить дополнительную информацию по электронной почте

## Примеры и кейсы

### Пример 1: Регистрация обращения по электронной почте

**Ситуация:** Клиент отправил письмо на support@pass24online.ru с описанием проблемы при входе в систему.

**Действия:**
1. Письмо получено в почтовом ящике
2. Создано обращение в системе с ID #12345
3. Отправлено автоматическое уведомление клиенту с ID обращения
4. Клиент получил уведомление и сохранил ID для дальнейшей переписки

**Результат:** Обращение успешно зарегистрировано, клиент получил подтверждение

**Выводы:** Важно убедиться, что клиент получил уведомление с ID, чтобы избежать проблем с дальнейшей коммуникацией

---

### Пример 2: Регистрация обращения через онлайн-форму

**Ситуация:** Клиент заполнил форму на сайте с вопросом о настройке продукта.

**Действия:**
1. Обращение автоматически создано в системе при отправке формы
2. Система присвоила ID #12346
3. Автоматически отправлено уведомление клиенту
4. Специалист техподдержки получил уведомление о новом обращении

**Результат:** Обращение зарегистрировано автоматически, обработка начата

**Выводы:** Форма автоматизирует процесс регистрации, но требует проверки полноты информации

## Часто задаваемые вопросы (FAQ)

### Вопрос 1: Что делать, если клиент не получил уведомление о регистрации обращения?

**Ответ:** 
- Проверить, не попало ли письмо в спам
- Убедиться, что обращение действительно зарегистрировано в системе
- Если обращение зарегистрировано, но клиент не получил уведомление, отправить уведомление вручную
- Предложить альтернативный канал связи (телефон, чат)

**Связанные материалы:** [Работа с почтовыми проблемами](#особые-случаи)

---

### Вопрос 2: Нужно ли регистрировать обращения, поступившие в нерабочее время?

**Ответ:** 
- Да, все обращения должны быть зарегистрированы
- Обращения, поступившие в нерабочее время, обрабатываются в порядке поступления в начале следующего рабочего дня
- Время реакции отсчитывается с начала рабочего времени

---

### Вопрос 3: Как обрабатывать обращения с некорректно сформулированными вопросами?

**Ответ:**
- Зарегистрировать обращение
- Присвоить ID
- Отправить уведомление о регистрации
- Запросить уточняющую информацию для понимания проблемы
- При необходимости связаться с клиентом по телефону для уточнения деталей

## Типичные ошибки и их решение

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Обращение не зарегистрировано в системе | Пропущено при проверке каналов связи | Регулярно проверять все каналы связи, использовать автоматические уведомления |
| Клиент не получил ID обращения | Проблемы с доставкой почты | Проверить статус отправки, отправить вручную, предложить альтернативный канал |
| Дублирование обращений | Клиент отправил обращение несколькими способами | Объединить обращения, использовать один ID, уведомить клиента |
| Неполная информация в обращении | Клиент не указал необходимые данные | Запросить дополнительную информацию, используя шаблон запроса |

## Критерии успеха

**Успешное выполнение определяется следующими критериями:**

- Обращение зарегистрировано в системе в течение 15 минут после поступления
- Клиент получил уведомление с уникальным ID обращения
- Вся доступная информация из обращения зафиксирована в системе
- Обращение готово для дальнейшей обработки (классификации и приоритизации)
