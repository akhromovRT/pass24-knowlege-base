## Краткое описание

Ведение реестра подключённых объектов: Реестр объектов (облако) в Б24 и файл «Подключенные объекты». Правила заполнения полей, источники данных, связь с задачей при «Счет оплачен». Содержит структуру полей и правила при расторжении/заморозке.

## Основной контент

### Процесс ведения Реестра подключенных объектов в компании

Есть два актуальных файла, по которым можно получить актуальную информацию по количеству подключенных объектов.

**Среди них:**
**1. Реестр объектов (облако)** на корпоративном портале Б24 (во вкладке * Автоматизация=>Списки=>Реестр объектов (облако)* ; чтобы посмотреть актуальное количество объектов, необходимо выбрать фильтр  *"Статус"=>"*  ***Активен*** *"*
**2. Файл "Подключенные объекты" **на диске группы *"Сопровождение клиентов". *Вспомогательный инструмент для ведения реестра.

**П. 1: Правила внесения записи в Реестр объектов (облако):**

Реестр заполняется на основании внесенной менеджером по продажам информации в автоматически созданной задаче **Добавление нового клиента/объекта в реестр Б24 и в 1С** при переходе сделки в воронке Облако на этап "Счет оплачен".

Информация в Реестре объектов (облака) по конкретному объекту должна соответствовать информации по тому же объекту в файле "Подключенные объекты" по аналогичным пунктам.

* **ID ** - автоматически присваивается системой
* **Название объекта ** - логически понятное название. Оно может не соответствовать названию объекта, которое присвоено в облаке.
* **Тип объекта ** - выбором из раскрывающегося списка.
* **Статус ** - выбором из раскрывающегося списка.
* **Юридическое лицо** - привязка к элементу CRM Компания
* **Ссылка на облако** - присваивается при создании Облака для клиента. Гиперссылка.
* **№ объекта в облаке** - присваивается при создании Адреса/объекта в облаке клиента.
* **Оборудование ** - выбором из списка
* **Паспорт объекта ** - вкладывается файл паспорта объекта, если оборудование установлено.
* **Адрес** - из карточки объекта в Яндексе
* **Город/регион** - Москва, Санкт-Петербург как города. Остальные как области. Клиент из Кемерово будет указан как клиент из Кемеровской области.
* **Локация на Яндекс карте ** - внесение координат
* **Комментарий ** - любая полезная информация.
* **Разделы ** - системное поле

При расторжении или заморозке объекта по нему устанавливается соответствующий статус (см. [Процесс расторжения и приостановки договора с клиентом](./Процесс%20расторжения%20и%20приостановки%20договора%20с%20клиентом.md)).

**П.2: Правила заполнения файла "Подключенные объекты":**

Файл заполняется на основании внесенной менеджером по продажам информации в автоматически созданной задаче **Добавление нового клиента/объекта в реестр Б24 и в 1С** при переходе сделки в воронке Облако на этап "Счет оплачен".

* **Город/регион** - Москва, Санкт-Петербург как города. Остальные как области. Клиент из Кемерово будет указан как клиент из Кемеровской области.
* **Тип объекта** - выбором из раскрывающегося списка.
* **Название объекта** - логически понятное название. Оно может не соответствовать названию объекта, которое присвоено в облаке.
* **Ссылка на облако** - присваивается при создании Облака для клиента.
* **№ объекта в облаке ** - присваивается при создании Адреса/объекта в облаке клиента.
* **Адрес ** - из карточки объекта в Яндексе
* **Ссылка на карту** - ссылка на карточку объекта в Яндекс
* **Широта ** - координаты широты из карточки объекта в Яндекс
* **Долгота ** - координаты долготы из карточки объекта в Яндекс
* **Сайт**
* **Название ЮЛ ** - сокращенное название ЮЛ, с которого поступают оплаты. Сначала наименование и потом форма. Пример: Ромашка ООО, не ООО "Ромашка"
* **Комментарий** - любая полезная информация (в т.ч. задача по расторжению в Б24, если такая информация есть)
* **Когда подключен** - первый оплаченный месяц. Если клиент внес деньги 31 августа, но за сентябрь, считаем его подключенным с сентября. Формат: месяц и год.

Расторгнутые объекты переносятся из вкладки "Подключенные объекты" на вкладку "Расторгнутые объекты" и указывается месяц и год расторжения.

*Таким образом, ведение Реестра объектов и Файла "Подключенных объектов" позволяет компании оперативно отслеживать все активные и прошлые объекты, управлять информацией о них и обеспечивать эффективное взаимодействие с клиентами.*

## Пошаговая инструкция

### Шаг 1: Получение задачи на добавление объекта

**Что делать:**
1. При переходе сделки в воронке Облако на этап "Счет оплачен" автоматически создаётся задача **"Добавление нового клиента/объекта в реестр Б24 и в 1С"**
2. Проверить наличие задачи в группе "Сопровождение клиентов"

**Где:** Битрикс24, воронка "Облако"

**Результат:** Получена задача на добавление объекта в реестр

---

### Шаг 2: Заполнение Реестра объектов (облако) в Б24

**Что делать:**
1. Открыть Реестр объектов (облако) в Б24 (Автоматизация=>Списки=>Реестр объектов (облако))
2. Создать новую запись
3. Заполнить все обязательные поля согласно правилам:
   - **ID** — автоматически присваивается системой
   - **Название объекта** — логически понятное название (может не соответствовать названию объекта в облаке)
   - **Тип объекта** — выбором из раскрывающегося списка
   - **Статус** — выбором из раскрывающегося списка (для новых объектов — "Активен")
   - **Юридическое лицо** — привязка к элементу CRM Компания
   - **Ссылка на облако** — присваивается при создании Облака для клиента (гиперссылка)
   - **№ объекта в облаке** — присваивается при создании Адреса/объекта в облаке клиента
   - **Оборудование** — выбором из списка
   - **Паспорт объекта** — вкладывается файл паспорта объекта, если оборудование установлено
   - **Адрес** — из карточки объекта в Яндексе
   - **Город/регион** — Москва, Санкт-Петербург как города. Остальные как области
   - **Локация на Яндекс карте** — внесение координат
   - **Комментарий** — любая полезная информация
   - **Разделы** — системное поле

**Где:** Битрикс24, Автоматизация=>Списки=>Реестр объектов (облако)

**Результат:** Объект добавлен в Реестр объектов (облако) в Б24

**Важно:** Информация в Реестре объектов (облака) по конкретному объекту должна соответствовать информации по тому же объекту в файле "Подключенные объекты" по аналогичным пунктам.

---

### Шаг 3: Заполнение файла "Подключенные объекты"

**Что делать:**
1. Открыть файл "Подключенные объекты" на диске группы "Сопровождение клиентов"
2. Добавить новую строку на вкладку "Подключенные объекты"
3. Заполнить все поля согласно правилам:
   - **Город/регион** — Москва, Санкт-Петербург как города. Остальные как области
   - **Тип объекта** — выбором из раскрывающегося списка
   - **Название объекта** — логически понятное название (может не соответствовать названию объекта в облаке)
   - **Ссылка на облако** — присваивается при создании Облака для клиента
   - **№ объекта в облаке** — присваивается при создании Адреса/объекта в облаке клиента
   - **Адрес** — из карточки объекта в Яндексе
   - **Ссылка на карту** — ссылка на карточку объекта в Яндекс
   - **Широта** — координаты широты из карточки объекта в Яндекс
   - **Долгота** — координаты долготы из карточки объекта в Яндекс
   - **Сайт**
   - **Название ЮЛ** — сокращенное название ЮЛ, с которого поступают оплаты. Сначала наименование и потом форма. Пример: Ромашка ООО, не ООО "Ромашка"
   - **Комментарий** — любая полезная информация (в т.ч. задача по расторжению в Б24, если такая информация есть)
   - **Когда подключен** — первый оплаченный месяц. Если клиент внес деньги 31 августа, но за сентябрь, считаем его подключенным с сентября. Формат: месяц и год

**Где:** Файл "Подключенные объекты" на диске группы "Сопровождение клиентов"

**Результат:** Объект добавлен в файл "Подключенные объекты"

**Важно:** Информация должна соответствовать информации в Реестре объектов (облако) в Б24.

---

### Шаг 4: Проверка соответствия данных

**Что делать:**
1. Сверить данные между Реестром объектов (облако) в Б24 и файлом "Подключенные объекты"
2. Убедиться, что все поля заполнены корректно
3. Проверить соответствие названия объекта, типа объекта, адреса, координат

**Результат:** Данные в обоих реестрах соответствуют друг другу

## Типичные ошибки и их решение

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Несоответствие данных между реестрами | Данные заполнялись вручную без сверки | Сверить данные между Реестром объектов (облако) в Б24 и файлом "Подключенные объекты" |
| Неправильный формат города/региона | Не учтено правило: Москва и СПб — города, остальные — области | Исправить формат: для Кемерово указать "Кемеровская область" |
| Неправильный формат названия ЮЛ | Не учтено правило: сначала наименование, потом форма | Исправить: "Ромашка ООО" вместо "ООО Ромашка" |
| Неправильная дата подключения | Не учтено правило: считать по оплаченному месяцу, а не по дате оплаты | Исправить: если оплата 31 августа за сентябрь, указать "сентябрь 2024" |

## Критерии успеха

**Успешное выполнение определяется следующими критериями:**
- Объект добавлен в Реестр объектов (облако) в Б24 со всеми обязательными полями
- Объект добавлен в файл "Подключенные объекты" со всеми обязательными полями
- Данные в обоих реестрах полностью соответствуют друг другу
- Все поля заполнены в соответствии с правилами
- Статус объекта установлен как "Активен" для новых объектов
