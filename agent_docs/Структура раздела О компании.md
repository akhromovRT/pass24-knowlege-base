# Предложенная структура раздела «О компании»

**Версия:** 1.0  
**Дата:** 2026-01-27  
**Назначение:** Конечная, полноценная структура раздела «О компании» с учётом рекомендаций, типов документов и лучших практик корпоративных баз знаний.

---

## 1. Принципы и границы раздела

### Назначение раздела «О компании»

- **Кросс-функциональная информация** — общая для всех сотрудников, не привязанная к одному отделу (ОП, ОС, Тех. поддержка, Маркетинг).
- **Административные и вспомогательные процедуры** — бухгалтерия, кадры, IT, внутренний регламент.
- **Онбординг и адаптация** — первый рабочий день, доступы, что изучить в базе знаний.
- **Обзор компании** — миссия, ценности, продукты (высокоуровнево), организационная структура (при необходимости).

### Чего нет в «О компании»

- **Стратегия 2026** — тематически отдельный раздел (стратегические документы, KPI, реализация). В главном README базы знаний представлен как отдельный раздел. Рекомендуется хранить отдельно от «О компании» (на верхнем уровне) или явно выделять как подраздел с пометкой «отдельный тематический раздел».
- **Отраслевые регламенты** — процессы продаж (ОП), сопровождения (ОС), техподдержки остаются в своих разделах. В «О компании» только ссылки на них.

### Учёт лучших практик

- **Единый источник истины** — каждая тема в одном месте, без дублирования.
- **Ориентация на пользователя** — документы для новичков, бухгалтерии, администраторов и т.д. с явной целевой аудиторией.
- **Структурированность** — иерархия папок, README в каждом подразделе, понятные названия.
- **Полнота без перегруза** — раздел покрывает всё необходимое для работы «о компании» и админки, но не подменяет операционные разделы.

## 3. Описание подразделов

### 3.1. 01 Обзор компании

| Элемент | Описание |
|--------|----------|
| **Назначение** | Общая информация о компании для всех сотрудников и новичков. |
| **Тип документов** | Справочник. |
| **Ключевой документ** | **Обзор компании.md** — миссия, ценности, кратко о продуктах (PASS24.Cloud и др.), история (при необходимости). Опционально: оргструктура, контакты ключевых лиц. |
| **Целевая аудитория** | Все сотрудники, в первую очередь новые. |
| **Связи** | Ссылки на ОП (продукты), ОС, Стратегия 2026 (цели и фокус). |

---

### 3.2. 02 Адаптация и онбординг

| Элемент | Описание |
|--------|----------|
| **Назначение** | Быстрый старт новых сотрудников: первый день, доступы, что изучить в базе знаний. |
| **Тип документов** | Обучение, процедура, чек-лист. |
| **Документы** | **Чек-лист адаптации нового сотрудника.md** — обязательный чек-лист (оформление, доступы, изучение ключевых разделов БЗ). **Первый рабочий день и доступы.md** — пошаговая инструкция (опционально, если не покрыто чек-листом). |
| **Целевая аудитория** | Новые сотрудники, HR, руководители. |
| **Связи** | Ссылки на Обзор компании, ОП/ОС/Тех. поддержка (порядок изучения), IT и рабочие места (доступы). |

---

### 3.3. 03 Бухгалтерия *(существующий)*

| Элемент | Описание |
|--------|----------|
| **Назначение** | Регламенты и процедуры бухгалтерии: ЭДО, 1С, согласование и подписание документов. |

| **Целевая аудитория** | Бухгалтерия, сотрудники, подписывающие документы. |

---

### 3.4. 04 Административные процедуры

| Элемент | Описание |
|--------|----------|
| **Назначение** | Внутренние административные правила: рабочий день, удалёнка, отпуска, больничные, отгулы. |
| **Тип документов** | Регламент, процедура. |
| **Документы** | **Регламент рабочего времени и удалённой работы.md** — график, удалёнка, коммуникация. **Отпуска, больничные и отгулы.md** — порядок согласования, оформления (опционально, если не в HR-системе). |
| **Целевая аудитория** | Все сотрудники, руководители, HR. |
| **Связи** | При необходимости — ссылки на Бухгалтерию (оплата), HR/кадры. |

### 3.5. 05 IT и рабочие места *(опционально)*

| Элемент | Описание |
|--------|----------|
| **Назначение** | Выдача техники, корпоративная почта, доступы (1С, Битрикс24, облако, VPN и т.д.). |
| **Тип документов** | Процедура, справочник. |

| **Связи** | Адаптация (доступы в первый день), Бухгалтерия (1С). |

## 4. Стратегия 2026

- В **главном README** базы знаний «Стратегия 2026» указана как **отдельный раздел**.
- Сейчас стратегия физически лежит в `01 О компании / 06 Стратегия 2026`.
- **Рекомендация:** рассматривать «Стратегию 2026» как **отдельный тематический раздел**. Варианты:
  - **А:** Вынести в `База знаний / Стратегия 2026 /` на верхний уровень и обновить ссылки в главном README.
  - **Б:** Оставить в `01 О компании / 06 Стратегия 2026`, но в README «О компании» явно указать, что это отдельный тематический раздел, и дать ссылку без дублирования его документов в общем списке «О компании».

В предложенной структуре **раздела «О компании»** подразделы 01–05 не включают Стратегию.

## 5. Типы документов и шаблон

- Все новые документы оформляются по **шаблону статьи** (`agent_docs/Шаблон статьи базы знаний.md`) и **инструкции** (`agent_docs/Инструкция по работе с базой знаний.md`).
- Обязательно: метаданные, целевая аудитория, краткое описание, основной контент, история изменений.
- Для процедур и регламентов: пошаговая инструкция, контрольный чек-лист.
- Отдел в метаданных: **О компании**.

---

## 6. Правило обновления README

- При добавлении, изменении или удалении документов в любом подразделе «О компании» обновлять соответствующий **README.md** (подраздела и при необходимости раздела).
- README раздела «О компании» должен отражать актуальный список подразделов и документов.

## 7. Пошаговое внедрение

1. **Согласовать** данную структуру с ответственным за раздел.
2. **Создать** недостающие подразделы (01 Обзор, 02 Адаптация, 04 Административные процедуры, при необходимости 05 IT).
3. **Добавить** README в каждый новый подраздел.
4. **Создать** документы по шаблону (сначала минимально необходимый набор: Обзор компании, Чек-лист адаптации, при необходимости регламенты админпроцедур и IT).
5. **Обновить** README раздела «О компании» под целевую структуру.
6. **Решить** вопрос размещения «Стратегия 2026» (вынести или оставить с явной пометкой в README).
